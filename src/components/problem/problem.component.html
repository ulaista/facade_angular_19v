The Problem: Tangled Dependencies

Imagine a component that needs to manage a user's session. It might need to interact with multiple services for authentication, data fetching, and logging. This creates tight coupling and makes the component complex.

Consequences:

  • High Complexity: The component has to coordinate calls between all services.
  • Poor Readability: Business logic is mixed with low-level service interactions.
  • Difficult to Test: You need to mock multiple dependencies to test the component.
  • Brittle: Changing one service might require changes in every component that uses it.
